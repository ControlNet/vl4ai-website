This project focuses on scene-understanding computer vision tasks including object detection, segmentation (instance, semantic, and panoptic), and depth estimation or completion from image, video, and point-cloud sequences using supervised, semi-supervised, few-shot, and self-supervised learning techniques.

Representative legacy publication links preserved from the old research page:

- **Image classification:** [Energy-based Self-Training and Normalization for Unsupervised Domain Adaptation](https://openaccess.thecvf.com/content/ICCV2023/papers/Herath_Energy-based_Self-Training_and_Normalization_for_Unsupervised_Domain_Adaptation_ICCV_2023_paper.pdf), [ProtoCon](https://arxiv.org/pdf/2303.13556.pdf), and [LAVA](https://openaccess.thecvf.com/content/WACV2023/papers/Nassar_LAVA_Label-Efficient_Visual_Learning_and_Adaptation_WACV_2023_paper.pdf).
- **Object detection:** [Knowledge Combination to Learn Rotated Detection Without Rotated Annotation](https://arxiv.org/pdf/2304.02199.pdf), [Learn to predict sets using feed-forward neural networks](https://arxiv.org/pdf/2001.11845.pdf), and [Generalized Intersection over Union](https://openaccess.thecvf.com/content_CVPR_2019/papers/Rezatofighi_Generalized_Intersection_Over_Union_A_Metric_and_a_Loss_for_CVPR_2019_paper.pdf).
- **Semantic segmentation:** [AerOSeg++](https://dl.acm.org/doi/epdf/10.1145/3787522), [Unsupervised Image Segmentation by Mutual Information Maximization and Adversarial Regularization](https://arxiv.org/pdf/2107.00691.pdf), and [Meta Learning with Differentiable Closed-form Solver for Fast Video Object Segmentation](https://arxiv.org/pdf/1909.13046.pdf).
- **Optical flow and depth:** [Unifying Flow, Stereo and Depth Estimation](https://arxiv.org/pdf/2211.05783.pdf), [GMFlow](https://arxiv.org/pdf/2111.13680.pdf), and [TopNet](https://openaccess.thecvf.com/content_CVPR_2019/papers/Tchapmi_TopNet_Structural_Point_Cloud_Decoder_CVPR_2019_paper.pdf).
- **Metrics and evaluation:** [How trustworthy are the existing performance evaluations for basic vision tasks?](https://arxiv.org/pdf/2008.03533.pdf)
